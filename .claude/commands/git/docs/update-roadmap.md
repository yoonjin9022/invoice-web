완료된 Task를 `docs/ROADMAP.md`에 반영합니다.

## 인자

`$ARGUMENTS` — 완료 처리할 Task 번호 (공백 구분, 복수 가능). 예: `009`, `009 010`, `009 010 011`

인자가 없으면 현재 ROADMAP.md를 읽고 완료 처리할 Task 번호를 사용자에게 물어보세요.

## 작업 절차

1. **`docs/ROADMAP.md` 파일 읽기**

2. **지정된 Task 번호별로 아래 작업 수행**

   a. Task 헤더에 `-- 완료` 추가 (이미 있으면 스킵)
   - 변경 전: `- **Task 009: 노션 API 연동 및 데이터 조회 구현** - 우선순위`
   - 변경 후: `- **Task 009: 노션 API 연동 및 데이터 조회 구현** -- 완료`

   b. 해당 Task 블록 내 체크박스 전체 체크
   - 변경 전: `- [ ] 항목 설명`
   - 변경 후: `- [x] 항목 설명`

3. **Phase 완료 여부 자동 판단**

   해당 Task가 속한 Phase의 모든 Task가 `-- 완료`로 표시되어 있으면:
   - Phase 헤더에 `-- 완료` 추가
   - 변경 전: `### Phase 3: 핵심 기능 구현`
   - 변경 후: `### Phase 3: 핵심 기능 구현 -- 완료`

4. **진행 현황 요약 테이블 업데이트**

   테이블의 완료 Task 카운트와 Phase 상태 컬럼을 업데이트:
   - 완료된 Task 수 재계산하여 `완료 Task` 컬럼 갱신
   - Phase 전체 완료 시 `상태` 컬럼을 `완료`로 변경
   - Phase 일부 완료 시 `상태` 컬럼을 `진행 중`으로 변경
   - `MVP 합계` 행도 함께 갱신

5. **"다음 작업 (Next Up)" 섹션 업데이트**

   완료된 Task를 목록에서 제거하고, 다음 미완료 Task를 순서대로 나열

6. **변경 내용 요약 출력**

   완료 처리된 Task 목록과 업데이트된 진행 현황을 간략히 출력

## 주의사항

- `-- 완료`가 이미 표시된 Task는 스킵
- Task 번호는 3자리 0-패딩 기준으로 매칭 (`9` 입력 시 `009`로 처리)
- 체크박스는 해당 Task 블록 내의 항목만 변경 (다른 Task 블록 영향 없음)
