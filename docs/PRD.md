# 노션 인보이스 웹뷰어 PRD

> 1인 개발자용 · 현재 구현 기준 · 즉시 개발 가능

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 노션 인보이스 웹뷰어 |
| **핵심 문제** | 노션에서 작성한 견적서를 클라이언트에게 공유하려면 노션 접근 권한 부여 또는 별도 PDF 변환이 필요해 불편함 |
| **솔루션** | 노션 데이터베이스와 연동하여 견적서별 고유 웹 URL을 자동 생성, 클라이언트가 링크만으로 확인 및 PDF 다운로드 가능. 발행자는 비밀번호 인증 후 관리자 대시보드에서 견적서를 관리 |
| **타겟 사용자** | 노션으로 견적서를 관리하는 1인 프리랜서 및 소규모 사업자 (발행자), 견적서를 수령하는 클라이언트 (수신자) |
| **핵심 가치** | 노션 작성 즉시 전문적인 견적서 웹 페이지 생성 및 PDF 다운로드, 발행자 전용 인증 대시보드로 견적서 통합 관리, 별도 설정 없이 공유 드롭다운으로 클라이언트에게 즉시 전달 |

## 사용자 여정

```
[발행자 여정]

1. 로그인 페이지
   ↓ 비밀번호 입력 후 로그인 버튼 클릭

2. 관리자 대시보드
   ↓ 전체 견적서 통계 확인 (건수, 상태별, 총 매출)

3. 견적서 목록 페이지 (사이드바 또는 대시보드에서 이동)
   ↓ 견적서 카드 클릭 또는 공유 드롭다운 사용

   [공유 드롭다운 선택]
     링크 복사 → 클립보드에 URL 저장
     이메일로 공유 → 이메일 앱 열기 (URL 포함)
     새 탭 미리보기 → 클라이언트 시점으로 확인

4. 견적서 상세 페이지 (발행자 접근)
   ↓ 내용 확인 후 공유 드롭다운으로 클라이언트에게 URL 전달

5. URL을 클라이언트에게 공유 (이메일, 메신저 등)


[클라이언트 여정]

1. 공유받은 URL 클릭
   ↓ 별도 로그인 없이 접근 (견적서 상세 페이지는 공개)

2. 견적서 상세 페이지
   ↓ 견적 내용 확인

   내용 확인 완료 → PDF 다운로드 버튼 클릭
   ↓

3. PDF 파일 로컬 저장 완료
```

## 기능 명세

### 1. 핵심 기능

| ID | 기능명 | 설명 | 필수 이유 | 관련 페이지 |
|----|--------|------|----------|------------|
| **F001** | 노션 데이터베이스 연동 | 노션 API를 통해 견적서 데이터베이스를 읽어와 웹에 반영 | 서비스의 데이터 소스이자 핵심 연동 기능 | 견적서 목록 페이지, 견적서 상세 페이지, 관리자 대시보드 |
| **F002** | 견적서 목록 조회 | 노션 DB의 모든 견적서를 리스트로 표시 (발행자용) | 발행자가 발행한 견적서를 한눈에 파악 가능 | 견적서 목록 페이지 |
| **F003** | 견적서 상세 웹뷰 렌더링 | 노션 페이지 ID 기반 고유 URL로 견적서 내용을 전문적인 UI로 렌더링 | 클라이언트가 링크만으로 견적서 확인 가능한 핵심 기능 | 견적서 상세 페이지 |
| **F004** | PDF 다운로드 | 렌더링된 견적서를 서버사이드에서 PDF로 변환하여 다운로드 | 클라이언트의 보관 및 출력 니즈 충족, 핵심 부가가치 | 견적서 상세 페이지 |
| **F005** | 관리자 대시보드 | 견적서 통계 (전체 건수, 상태별 건수, 총 매출) 및 최근 발행 견적서 5건 미리보기 | 발행자가 비즈니스 현황을 한눈에 파악 | 관리자 대시보드 |
| **F006** | 발행자 인증 | 비밀번호 로그인으로 관리자 라우트 보호, HTTP-only 세션 쿠키 기반 인증 | 관리자 기능을 발행자만 접근하도록 보호 | 로그인 페이지, 관리자 대시보드, 견적서 목록 페이지 |
| **F007** | 공유 드롭다운 | 링크 복사, 이메일로 공유, 새 탭 미리보기 옵션이 포함된 공유 메뉴 | 다양한 방식으로 클라이언트에게 견적서 URL 전달 가능 | 견적서 목록 페이지, 견적서 상세 페이지 |

### 2. 지원 기능

| ID | 기능명 | 설명 | 필수 이유 | 관련 페이지 |

| **F012** | 견적서 없음 처리 | 잘못된 URL 또는 존재하지 않는 견적서 접근 시 안내 페이지 표시 | 클라이언트가 잘못된 링크 접근 시 혼란 방지 | 오류 페이지 |

## 데이터 모델

> 노션 데이터베이스를 단일 진실 공급원(Source of Truth)으로 사용하므로 별도 DB는 없음.
> 아래는 노션에서 읽어온 데이터를 앱에서 사용하는 타입 정의 및 인증 관련 설정.

### Invoice (견적서)

> 실제 노션 Invoices DB 컬럼 기준 (2026-02 확인)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 노션 페이지 ID (고유 식별자) | string (UUID 형식) |
| title | 견적서 번호 (견적서 번호 속성, 예: INV-2025-001) | string |
| status | 견적서 상태 | 'pending' \| 'sent' \| 'approved' \| 'completed' |
| clientName | 클라이언트명 | string |
| issuedAt | 발행일 | Date |
| validUntil | 유효기간 | Date (optional) |
| items | 견적 품목 목록 (Items DB relation) | InvoiceItem[] |
| totalAmount | 총 금액 (KRW, 노션 DB에서 직접 읽음) | number |
| notes | 비고 | string (optional) |
| notionPageUrl | 노션 원본 페이지 URL | string |
| createdAt | 노션 페이지 생성일시 | Date |
| updatedAt | 노션 페이지 최종 수정일시 | Date |

### InvoiceItem (견적 품목)

> 실제 노션 Items DB 컬럼 기준 (2026-02 확인)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 품목 고유 식별자 (노션 페이지 ID) | string |
| invoiceId | 소속 견적서 ID | → Invoice.id |
| name | 항목명 | string |
| quantity | 수량 | number |
| unitPrice | 단가 | number |
| amount | 금액 (quantity × unitPrice) | number |

### AdminSession (발행자 인증 세션)

> 서버사이드에서 HTTP-only 쿠키로 관리. 별도 DB 저장 없이 HMAC-SHA256 서명 검증 방식 사용.

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| token | HMAC-SHA256 서명된 세션 토큰 | string (HTTP-only 쿠키) |
| expiresAt | 세션 만료일시 | 7일 유효기간 |
| secure | HTTPS 전용 여부 | boolean (프로덕션: true) |

### NotionDatabaseConfig (노션 연동 설정)

> 환경변수로 관리되며, 코드 내 타입 참조용으로만 사용.

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| apiKey | 노션 통합(Integration) API 키 | string (환경변수: NOTION_API_KEY) |
| databaseId | 견적서 데이터베이스 ID | string (환경변수: NOTION_DATABASE_ID) |

## 주요 구현 사항

### 인증 흐름

```
발행자 접근 시도 (관리자 라우트)
  └─ proxy.ts에서 세션 쿠키 검증
      ├─ 유효한 세션 → 요청 통과
      └─ 세션 없음/만료 → /login으로 307 리다이렉트

로그인 프로세스
  └─ 로그인 페이지에서 비밀번호 입력
  └─ loginAction (서버 액션) 호출
  └─ HMAC-SHA256으로 세션 토큰 생성
  └─ HTTP-only 쿠키 저장 (7일 유효)
  └─ /dashboard로 리다이렉트
```

### 노션 API 연동 방식

```
Server Component (견적서 상세 페이지)
  └─ getInvoice(id) — React cache()로 래핑된 함수
  └─ notionClient.databases.query()  →  견적서 목록
  └─ notionClient.pages.retrieve()   →  견적서 상세
  └─ 응답 데이터를 Invoice 타입으로 변환 (mapNotionPageToInvoice)
  └─ ISR revalidate: 300 적용
  └─ JSX 렌더링
```

- 노션 API 호출은 서버 컴포넌트에서만 수행 (API 키 노출 방지)
- `NOTION_API_KEY`, `NOTION_DATABASE_ID`는 `.env.local`에서 관리
- Vercel 배포 시 환경변수 대시보드에서 설정

### PDF 다운로드 방식

- **서버사이드 생성**: API Route에서 `@react-pdf/renderer`로 PDF Buffer 생성 후 스트리밍
- PDF 레이아웃은 웹 뷰와 별도 컴포넌트로 관리하여 일관된 디자인 유지
- 다운로드 파일명: `invoice-{클라이언트명}-{발행일}.pdf`
- 항상 라이트 테마로 렌더링 (다크모드 설정과 무관)

### URL 구조 (페이지명 기준)

- 로그인 페이지: 비인증 발행자 진입점
- 관리자 대시보드: 로그인 후 메인 페이지 (인증 필요)
- 견적서 목록 페이지: 관리자 라우트 그룹 내 (인증 필요)
- 견적서 상세 페이지: 노션 페이지 ID 기반 동적 라우트 (공개)
- PDF API Route: 서버사이드 PDF 생성 엔드포인트 (공개)
- Next.js App Router의 라우트 그룹 `(admin)` 활용
