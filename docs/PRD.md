# 노션 인보이스 웹뷰어 PRD

> 1인 개발자용 · MVP 범위 · 즉시 개발 가능

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 노션 인보이스 웹뷰어 |
| **핵심 문제** | 노션에서 작성한 견적서를 클라이언트에게 공유하려면 노션 접근 권한 부여 또는 별도 PDF 변환이 필요해 불편함 |
| **솔루션** | 노션 데이터베이스와 연동하여 견적서별 고유 웹 URL을 자동 생성, 클라이언트가 링크만으로 확인 및 PDF 다운로드 가능 |
| **타겟 사용자** | 노션으로 견적서를 관리하는 1인 프리랜서 및 소규모 사업자 (발행자), 견적서를 수령하는 클라이언트 (수신자) |
| **핵심 가치** | 노션 작성 즉시 전문적인 견적서 웹 페이지 생성 및 PDF 다운로드, 별도 설정 없이 URL 공유만으로 클라이언트에게 전달 |

## 사용자 여정

```
[발행자 여정]

1. 노션 데이터베이스
   ↓ 견적서 항목 입력 (클라이언트명, 품목, 금액 등)

2. 노션 페이지 저장
   ↓ 자동으로 고유 인보이스 ID 생성 (노션 페이지 ID 활용)

3. 견적서 상세 페이지 (웹)
   ↓ 미리보기 확인 후 URL 복사

4. URL을 클라이언트에게 공유 (이메일, 메신저 등)


[클라이언트 여정]

1. 공유받은 URL 클릭
   ↓ 별도 로그인 없이 접근

2. 견적서 상세 페이지
   ↓ 견적 내용 확인

   내용 확인 완료 → PDF 다운로드 버튼 클릭
   ↓

3. PDF 파일 로컬 저장 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 노션 데이터베이스 연동 | 노션 API를 통해 견적서 데이터베이스를 읽어와 웹에 반영 | 서비스의 데이터 소스이자 핵심 연동 기능 | 견적서 목록 페이지, 견적서 상세 페이지 |

| **F003** | 견적서 상세 웹뷰 렌더링 | 노션 페이지 ID 기반 고유 URL로 견적서 내용을 전문적인 UI로 렌더링 | 클라이언트가 링크만으로 견적서 확인 가능한 핵심 기능 | 견적서 상세 페이지 |
| **F004** | PDF 다운로드 | 렌더링된 견적서 웹 페이지를 PDF 파일로 변환하여 다운로드 | 클라이언트의 보관 및 출력 니즈 충족, 핵심 부가가치 | 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 노션 API 연결 설정 | Notion API 키 및 데이터베이스 ID를 환경변수로 관리하여 연결 | 노션 데이터 연동을 위한 전제 조건 | (환경변수 설정, 페이지 없음) |

## 데이터 모델

> 노션 데이터베이스를 단일 진실 공급원(Source of Truth)으로 사용하므로 별도 DB는 없음.
> 아래는 노션에서 읽어온 데이터를 앱에서 사용하는 타입 정의.

### Invoice (견적서)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 노션 페이지 ID (고유 식별자) | string (UUID 형식) |
| title | 견적서 제목 | string |
| status | 견적서 상태 | 'draft' \| 'sent' \| 'approved' \| 'completed' |
| clientName | 클라이언트(수신자) 이름 또는 회사명 | string |
| clientEmail | 클라이언트 이메일 | string (optional) |
| issuerName | 발행자 이름 또는 회사명 | string |
| issuerEmail | 발행자 이메일 | string (optional) |
| issuedAt | 발행일 | Date |
| validUntil | 견적 유효기간 | Date (optional) |
| items | 견적 품목 목록 | InvoiceItem[] |
| subtotal | 소계 (세전 합산) | number |
| taxRate | 세율 (예: 0.1 = 10%) | number |
| taxAmount | 세액 | number |
| totalAmount | 최종 합계 (세후) | number |
| currency | 통화 단위 | string (기본값: 'KRW') |
| notes | 비고 사항 | string (optional) |
| notionPageUrl | 노션 원본 페이지 URL | string |
| createdAt | 노션 페이지 생성일시 | Date |
| updatedAt | 노션 페이지 최종 수정일시 | Date |

### InvoiceItem (견적 품목)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 품목 고유 식별자 | string |
| invoiceId | 소속 견적서 ID | → Invoice.id |
| name | 품목명 | string |
| description | 품목 상세 설명 | string (optional) |
| quantity | 수량 | number |
| unitPrice | 단가 | number |
| amount | 품목 합계 (quantity × unitPrice) | number |

### NotionDatabaseConfig (노션 연동 설정)

> 환경변수로 관리되며, 코드 내 타입 참조용으로만 사용.

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| apiKey | 노션 통합(Integration) API 키 | string (환경변수: NOTION_API_KEY) |
| databaseId | 견적서 데이터베이스 ID | string (환경변수: NOTION_DATABASE_ID) |

## 기술 스택

### 프론트엔드

- **Next.js 15** - App Router, Server Components (서버에서 노션 API 직접 호출)
- **React 19** - UI 라이브러리
- **TypeScript** - 타입 안전성
- **TailwindCSS v4** - 유틸리티 CSS
- **shadcn/ui** - UI 컴포넌트 라이브러리 (Table, Badge, Card, Button 등)

### 상태 관리

- **React Server Components** - 노션 API 데이터는 서버에서 fetch (Zustand 캐시 불필요)
- **React Hook Form + Zod** - 향후 폼 기능 추가 시 (MVP에서는 폼 없음)

### 노션 연동

- **@notionhq/client** - 공식 Notion JavaScript SDK
- **notion-to-md** - 노션 블록을 마크다운으로 변환 (비고/설명 렌더링용, 선택적 적용)

### PDF 생성

- **@react-pdf/renderer** - React 컴포넌트를 PDF로 렌더링 (서버사이드 생성 방식)
  - 또는 **html2canvas + jsPDF** - 클라이언트사이드 웹 페이지 캡처 방식 (구현 단순)
  - 권장: `@react-pdf/renderer`로 별도 PDF 레이아웃 컴포넌트 구성

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 패키지 관리

- **pnpm** - 빠른 의존성 관리

## 주요 구현 사항

### 노션 API 연동 방식

```
Server Component (견적서 상세 페이지)
  └─ notionClient.pages.retrieve(pageId)  →  노션 API
  └─ notionClient.blocks.children.list(pageId)  →  블록(품목) 조회
  └─ 응답 데이터를 Invoice 타입으로 변환
  └─ JSX 렌더링
```

- 노션 API 호출은 서버 컴포넌트에서만 수행 (API 키 노출 방지)
- `NOTION_API_KEY`, `NOTION_DATABASE_ID`는 `.env.local`에서 관리
- Vercel 배포 시 환경변수 대시보드에서 설정

### PDF 다운로드 방식

- **서버사이드 생성 (권장)**: API Route에서 `@react-pdf/renderer`로 PDF Buffer 생성 후 스트리밍
- PDF 레이아웃은 웹 뷰와 별도 컴포넌트로 관리하여 일관된 디자인 유지
- 다운로드 파일명: `invoice-{클라이언트명}-{발행일}.pdf`

### URL 구조 (페이지명 기준)

- 견적서 목록 페이지: 루트 경로
- 견적서 상세 페이지: 노션 페이지 ID를 경로 파라미터로 사용
- Next.js App Router의 동적 라우트 세그먼트 활용
